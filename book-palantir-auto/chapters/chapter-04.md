## 4.3 数据分类分级

### 4.3.1 数据分类方法

根据汽车主机厂的业务特点，建议采用多维度分类：

**1. 按业务领域分类**

- 研发类、采购类、生产类、质量类
- 销售类、售后类、财务类、人力类

**2. 按数据来源分类**

- 业务系统数据
- 设备/IoT数据
- 外部数据
- 衍生/计算数据

**3. 按数据结构分类**

- 结构化数据
- 半结构化数据（JSON、XML）
- 非结构化数据（文档、图片、视频）

### 4.3.2 数据分级标准

建议采用四级分类：

| 级别 | 定义 | 典型数据 | 保护要求 |
|------|------|---------|---------|
| L4 | 极敏感 | 客户个人信息、密钥 | 加密存储，严格授权 |
| L3 | 敏感 | 财务数据、技术机密 | 加密存储，授权审批 |
| L2 | 内部 | 业务数据、运营数据 | 授权访问，审计追溯 |
| L1 | 公开 | 产品目录、公开信息 | 基本保护 |

### 4.3.3 汽车行业特有数据

**重要数据识别**：

| 数据类型 | 所属级别 | 识别要点 |
|---------|---------|---------|
| 车辆VIN | L3 | 车辆身份标识，可关联个人 |
| 客户个人信息 | L4 | 姓名、电话、身份证等 |
| 车辆行驶数据 | L3 | 定位轨迹、驾驶行为 |
| 研发技术资料 | L3/4 | 产品配方、工艺参数 |
| 供应商报价 | L3 | 商业机密 |
| 生产工艺参数 | L3 | 企业核心资产 |
| 质量缺陷数据 | L2 | 可能涉及召回 |
| 投诉记录 | L2 | 客户隐私 |

## 4.4 数据资产目录建设

### 4.4.1 目录架构

```
数据资产目录
│
├─ 一级目录（业务领域）
│  ├─ 研发数据
│  ├─ 供应链数据
│  ├─ 生产制造数据
│  ├─ 质量数据
│  ├─ 销售售后数据
│  ├─ 经营管理数据
│  └─ 外部数据
│
├─ 二级目录（主题域）
│  例如：生产制造数据
│  ├─ 生产计划
│  ├─ 生产执行
│  ├─ 工艺数据
│  ├─ 设备数据
│  └─ 人员数据
│
├─ 三级目录（数据集）
│  例如：生产执行
│  ├─ 生产工单
│  ├─ 产出记录
│  ├─ 人员工时
│  └─ 工序追溯
│
└─ 数据项（字段）
    例如：生产工单
    ├─ 工单编号
    ├─ 物料编码
    ├─ 计划数量
    ├─ 实际数量
    └─ ...
```

### 4.4.2 目录元数据

每个数据资产应包含以下元数据：

| 元数据项 | 说明 | 示例 |
|---------|------|------|
| 数据集名称 | 中文业务名称 | 生产工单 |
| 数据集代码 | 英文技术名称 | pro_work_order |
| 数据源 | 来源系统 | MES |
| 数据更新频率 | 更新方式 | 实时 |
| 数据量级 | 数据规模 | 1000万条/天 |
| 数据敏感性 | 敏感级别 | L2 |
| 数据 Owner | 业务负责部门 | 制造部 |
| 数据 Stewards | 数据管理专员 | 张三 |
| 描述 | 业务说明 | 记录生产工单信息 |
| 标签 | 业务标签 | 生产、计划、核心 |

### 4.4.3 目录建设步骤

**第一步：现状调研**

- 访谈各业务部门
- 梳理现有系统清单
- 了解数据现状

**第二步：分类分级**

- 确定分类维度
- 执行分级标准
- 标识敏感数据

**第三步：目录设计**

- 设计目录结构
- 定义元数据规范
- 建立管理机制

**第四步：工具实施**

- 选择目录平台
- 导入元数据
- 发布目录

**第五步：持续运营**

- 定期更新
- 权限管理
- 质量监控
