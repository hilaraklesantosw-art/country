# Book Writer - 自动化书籍写作系统

## 概述

本skill提供一套完整的书籍写作流程，从大纲构思到最终成书，涵盖写作的各个阶段。

## 写作流程

### 阶段1：确定写作主题

首先与用户确认：
- **书籍主题**：要写什么类型的书？
- **目标读者**：谁会读这本书？
- **书籍规模**：预计多少字？短篇/中篇/长篇？
- **风格要求**：学术/通俗/小说/技术等

### 阶段2：编写大纲 (Outline)

根据确定的主题，撰写书籍大纲：

```
# [书名]

## 第一章：XXX
### 1.1 XXX
### 1.2 XXX

## 第二章：XXX
...
```

**要点**：
- 大纲应包含书籍的整体结构和章节目录
- 章节数量根据书籍规模确定（通常10-30章）
- 每个章节应有明确的主题

### 阶段3：编写详细提纲 (Detailed Outline)

基于大纲，为每个章节撰写详细提纲：

```
## 第一章：XXX

### 1.1 章节主题
- 要点1：...
- 要点2：...
- 要点3：...

### 1.2 章节主题
...
```

**要点**：
- 每个章节下的每个小节需要列出3-5个核心要点
- 要点应涵盖该小节的核心内容
- 确保各章节之间逻辑连贯

### 阶段4：撰写初稿 (First Draft)

按照提纲逐章节撰写初稿：

**写作规范**：
- 每章节独立成文件，便于修改
- 使用markdown格式
- 保持一致的写作风格
- 适当使用标题层级

**命名规范**：
```
book-draft/
├── chapter-01.md
├── chapter-02.md
├── chapter-03.md
...
```

### 阶段5：润色初稿 (Polishing)

进行多轮润色：

**第一轮 - 结构润色**：
- 检查章节逻辑顺序
- 确认内容完整无遗漏
- 调整章节过渡

**第二轮 - 内容润色**：
- 优化表达方式
- 统一术语
- 加强说服力

**第三轮 - 细节润色**：
- 检查语法错误
- 统一格式
- 添加图表说明

### 阶段6：整合成书 (Final Assembly)

将所有章节整合为完整书籍：

```
book-final/
├── README.md          # 书籍简介、目录
├── chapter-01.md
├── chapter-02.md
...
└── SUMMARY.md         # 完整目录
```

**最终检查清单**：
- [ ] 封面/简介完整
- [ ] 目录清晰
- [ ] 章节顺序正确
- [ ] 内容连贯
- [ ] 格式统一

## 写作技巧

### 保持一致性
- 定义统一的术语表
- 建立写作风格指南
- 使用固定的句式结构

### 提高效率
- 每天设定写作字数目标
- 先完成初稿，再进行修改
- 避免边写边改

### 保证质量
- 每章完成后休息一下再回头检查
- 找一个可信的读者提供反馈
- 大声朗读帮助发现语病

## 快速开始

当用户要求写书时，按照以下顺序执行：

1. **确认主题** - 询问用户书籍主题、目标读者等
2. **编写大纲** - 创建书籍结构框架
3. **编写提纲** - 扩展为详细内容大纲
4. **撰写初稿** - 按提纲填充内容
5. **润色修改** - 多轮优化
6. **整合交付** - 形成最终书籍

## 使用示例

**用户输入**：
> 帮我写一本关于Python编程的书

**执行流程**：
1. 确认：主题是Python编程，面向初学者，预计10万字
2. 大纲：包含基础语法、面向对象、常用库、项目实战等15章
3. 提纲：每章3-5节，每节3-5个要点
4. 初稿：逐章撰写，约5000字/章
5. 润色：进行3轮优化
6. 成书：整合为完整书籍，生成目录
