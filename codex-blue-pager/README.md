# 《Codex 蓝皮书》

副标题：产品形态、工作机制与真实开发协作指南

这是一本面向中文开发者、团队负责人和 AI 工具研究者的 Codex 研究型书稿。它不把 Codex 写成“神奇外挂”，也不把它简化成“一个会写代码的聊天框”，而是把它放回真实的软件开发现场，讨论它的能力、边界、机制、风险和长期使用方式。

本书采用“从浅到深”的结构，既适合第一次系统了解 Codex 的读者，也适合作为团队内部培训、选型评估和工作流设计的参考材料。

## 适合谁

- 想系统理解 Codex，而不是只会零散试用的人
- 想把 Codex 纳入真实开发流程的个人开发者
- 需要制定团队 AI 协作规范的技术负责人
- 正在做 AI 编程工具调研、对比和采购决策的人

## 阅读路径

- 想快速建立认知：第 1-2 章
- 想尽快上手：第 3-4 章
- 想做扩展和规范：第 5-6 章
- 想控制风险和形成长期方法：第 7-8 章

## 本书核心主张

- 理解 Codex，重点不在“它会不会写代码”，而在“它如何进入工程闭环”
- 用好 Codex，关键不在花哨提示词，而在上下文、约束、验证与复盘
- 团队级价值不来自偶发惊艳，而来自 AGENTS.md、Skills、MCP 和规则沉淀
- 真正决定上限的，不是模型参数本身，而是任务拆解和工作流设计能力
