# 术语表

## Agent

指能够围绕目标持续采取动作的 AI 协作体，而不是只输出一次回答的模型调用。

## Sandbox

用于约束代理可访问文件、命令和网络范围的执行边界。

## Approval

在代理执行高风险动作前或失败升级时引入的人类确认机制。

## Web Search

让代理访问外部网络信息的能力。它能提高信息时效性，但同时引入外部内容风险。

## AGENTS.md

面向 Codex 的仓库级或目录级规则文件，用于描述项目结构、常用命令、约束和验收要求。

## Skill

对重复流程、特定任务或领域知识的可复用封装，通常以说明文件加资源目录组成。

## MCP

一种让代理在受控前提下访问外部工具、服务和上下文的连接机制。

## Subagent

由主代理派生出的子任务代理，通常用于并行的信息收集、代码处理或验证工作。

## Worktree

Git 提供的多工作树机制，适合在同一仓库上并行处理多个任务分支。

## Cloud Task

在云端环境中执行的 Codex 任务，适合后台化、委托式和并行化场景。

## Reasoning Effort

模型在任务中投入的推理强度设置。通常越高，成本和时延越高，但复杂任务的稳定性也可能更好。

## Prompt Injection

外部内容通过“伪装成指令”的方式影响代理行为的风险类型，不局限于网页，也可能来自文档、评论或其他不可信输入。
