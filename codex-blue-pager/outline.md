# 《Codex 蓝皮书》目录与章节定位

## 第 1 章 认识 Codex

目标：建立正确认知。

解决的问题：

- Codex 是什么
- 为什么它值得关注
- 它和 ChatGPT、IDE 自动补全、普通 AI 编程助手分别有什么区别

## 第 2 章 Codex 的工作机制

目标：理解它为什么这样工作。

解决的问题：

- Agent 协作到底意味着什么
- 本地执行与云端执行的差异
- sandbox、approval、web search、AGENTS.md、skills、subagents 分别扮演什么角色

## 第 3 章 上手方式与入口选择

目标：帮助读者选对使用入口。

解决的问题：

- App、CLI、IDE、Cloud 适合什么人
- 首次安装、登录、项目选择和第一条任务应该怎么做

## 第 4 章 把 Codex 接进真实开发流程

目标：把工具使用从“聊天”推进到“交付”。

解决的问题：

- 怎么让 Codex 读项目、改代码、跑测试、做 review、提交 diff
- 怎么建立“分析 -> 修改 -> 验证”的闭环

## 第 5 章 Skills / MCP / 扩展能力

目标：把零散使用升级为可扩展体系。

解决的问题：

- Skills 是什么，什么时候值得写
- MCP 应该怎么理解，适合哪些连接场景
- 怎样把重复流程沉淀成可复用能力

## 第 6 章 模型、模式与配置

目标：建立配置与选型意识。

解决的问题：

- 不同模型与任务如何匹配
- 推理强度、审批策略、工作目录、自动化选项如何影响效果
- 哪些配置该统一，哪些配置该按项目区分

## 第 7 章 安全、边界与成本控制

目标：让读者知道什么能放手，什么不能。

解决的问题：

- 沙箱、审批、网络访问和敏感仓库的风险
- Prompt Injection、错误修改、过度信任的常见问题
- token 成本、云任务成本和团队使用成本如何控制

## 第 8 章 生态、对比与长期用法

目标：回答“为什么长期用”和“怎么长期用”。

解决的问题：

- Codex 和其他工具如何分工
- 什么团队适合引入 Codex
- 个人与团队如何沉淀 AGENTS.md、Skills、模板与规则
