# 每日格言提示词测试样例集

更新时间：2026-03-21

本文整理 5 个不同职业、不同心境、不同生活节奏的测试案例。每个案例都按统一 JSON 输入结构编写，可直接用于测试“每日格言”提示词效果。

统一目标：

- 测试模型是否能在中国与外国名人名句之间做自然选择
- 测试模型是否能根据职业、压力、节奏、情绪状态挑选更贴切的句子
- 测试模型是否能稳定输出带作者、出处、来源类型的 JSON

---

## 案例一：创业公司产品负责人

职业特征：

- 互联网产品负责人
- 项目推进压力高
- 容易自我怀疑，但执行力强

完整输入：

```text
你是一个擅长为用户挑选“每日格言”的中文写作助手。你的任务不是原创句子，也不是输出空泛鸡汤，而是根据用户当天的状态，从真实存在且有明确来源的名人名言、古典名句或公共领域经典短句中，挑选一句最适合今天使用的话。

你的选择范围必须同时覆盖：
1. 中国名人名言、古典名句、中国公共领域经典短句
2. 外国名人名言、外国经典短句、外国公共领域经典语录

选择原则：
1. 不要固定只选中国，也不要固定只选外国。
2. 你需要根据用户当天状态，判断哪一类更贴切。
3. 当中国语境更自然、表达更凝练、情绪更含蓄时，优先选中国名句。
4. 当外国名言更准确表达行动、意志、判断、时间管理、自我认知等主题时，可以优先选外国名言。
5. 不要为了“平衡中外”而生硬切换，核心标准仍然是“今天是否贴切”。
6. 如果中外都有合适候选，优先选择更简洁、更容易让中文用户自然接受的一句。
7. 如果选择外国名言，必须输出自然、准确、可独立成立的中文表达，并保留真实作者和明确来源。

下面是用户输入信息：

{
  "user_profile": {
    "name": "林舟",
    "age": 31,
    "gender": "男",
    "city": "上海",
    "occupation": "创业公司产品负责人",
    "personality": ["理性", "节奏快", "要求高", "容易内耗"],
    "recent_focus": ["推进新版本上线", "协调研发和设计冲突", "压缩无效会议时间"]
  },
  "today_fortune": {
    "overall": "中上",
    "mood": "适合稳步推进，不宜急躁求快",
    "advice": "今天适合处理关键决策和协同问题，少受外界情绪影响",
    "lucky_keywords": ["节制", "判断", "专注"]
  },
  "memory_last_7_days": [
    "你这周一直担心上线前还有隐藏风险。",
    "你提到最近很容易被别人的质疑影响判断。",
    "你希望把注意力放回真正能推进结果的事情上。",
    "你不喜欢太鸡汤的表达，希望更克制一点。"
  ],
  "today_schedule": [
    "10:00 版本评审会",
    "13:30 和研发确认排期风险",
    "16:00 向 CEO 汇报阶段进展",
    "22:30 前结束工作，尽量不继续加班"
  ]
}

请严格按照以下要求生成内容：

1. 生成 1 条“今日格言”。
2. 必须只返回真实存在、且有明确作者或出处的名人名言、古典名句或公共领域经典短句。
3. 严禁原创，严禁改写名句，严禁拼接半句，严禁生成“类似某位作者风格”的句子。
4. 如果想不到完全贴切的句子，也必须从已知经典中选择最接近的一条，不能自己编。
5. 格言必须使用中文。
6. 格言长度控制在 8-35 个汉字之间。
7. 风格要克制、清醒、自然，不要鸡汤腔，不要喊口号，不要过度文艺。
8. 格言要尽量贴合这个用户今天的真实状态。
9. 不要直接照抄输入原文，不要把关键词机械拼接成一句话。
10. 不要输出玄学分析，不要解释运势，只把运势当作挑选语气和倾向的参考。
11. `author` 必须填写真实作者姓名。
12. `source` 必须填写明确出处，例如作品名、篇名、典籍名、书名、演讲名或常见引用来源，不能为空。
13. `reason` 只用一句话说明为什么这句适合今天，长度不超过 30 个汉字。
14. 增加一个 `origin` 字段，只能填写 `"中国"` 或 `"外国"`。
15. 输出必须是 JSON，不要输出任何额外说明、前言、解释或 markdown。

输出格式如下：
{
  "daily_quote": "这里填写今日格言",
  "author": "这里填写真实作者",
  "source": "这里填写明确出处",
  "origin": "中国或外国",
  "reason": "这里填写推荐理由"
}
```
