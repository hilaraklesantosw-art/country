# 后记

如果你已经从前言一路读到这里，那说明你对 Codex 的期待，应该已经从“它能不能帮我做点什么”变成了“我怎样才能持续、稳定、成体系地把它用起来”。

这正是我希望这本书完成的转变。

我并不认为一本文字就能替代所有实践。真正让你拉开差距的，永远不是“看懂了一套方法”，而是“你有没有把这套方法用到下一个真实任务里”。所以读完这本书后，最重要的事不是收藏，而是立刻去做。

去找一个真实仓库，让 Codex 帮你读第一轮结构。

去找一个真实需求，让它参与一次拆解和交付。

去找一个真实报错，让它跟你一起完成一次定位、修改和验证。

去记录一次失败，然后把失败写成规则。

只要你真正做过几轮，你就会发现，Codex 最有价值的地方从来不是“替你思考”，而是“在你思考清楚之后，帮你更快推进、更稳验证、更好沉淀”。

如果这本书能够帮你建立这样的使用习惯，它就已经完成了自己的任务。

至于后面的路，最终还是要靠你在真实项目中不断迭代自己的方法。

如果你希望少走一些开通、使用和高频协作的弯路，附录里也保留了作者支持说明。是否需要，取决于你的时间成本，而不是这本书本身是否完整。

愿你读完这本书之后，不只是多了一个工具，而是多了一套真正可复用的工作方式。
