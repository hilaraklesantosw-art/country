# 前言

写这本书的念头，来自一个很直接的观察。

这两年，越来越多人开始接触 AI 编程工具。有人因为一段演示视频感到兴奋，有人因为几次高质量回答开始上头，也有人在试了几次之后迅速失望，得出一个含糊的结论：好像有点用，但也没那么神。

我越来越觉得，这里面最核心的问题，不是工具本身，而是使用者和工具之间的位置关系没有摆正。

很多人把 Codex 当成万能替身，希望它直接接手工作；也有人把它当成更聪明一点的搜索框，始终只停留在问答层面。前者容易失望，后者很难提效。真正高效的路径，其实在两者之间：把 Codex 当成一个可以进入真实开发任务、但必须被正确约束和正确验证的协作者。

这本书就是围绕这个认识写成的。

我不打算把它写成一本概念罗列手册，也不想把它写成一本只会堆砌案例的速成读物。相比“告诉你 Codex 很强”，我更关心的是另一件事：**怎样让你在现实开发环境中稳定地把它用起来。**

所以你会发现，这本书反复强调几件事情：

- 任务要说清楚
- 上下文要给够
- 边界要明确
- 验证不能省略
- 失败要复盘

这些东西听起来甚至不够“性感”，但它们恰恰是真正能把工具使用拉开差距的部分。

我也没有回避一个现实问题：当你真的开始高频使用 Codex，会员、稳定性、时间成本和配套支持都会变成实际问题。与其假装这些问题不存在，不如坦诚地把它们摆出来。所以在本书中，我选择用透明而克制的方式处理相关服务说明。正文优先交付价值，附录再承接有需求的读者。

如果你拿起这本书，只是想看一眼“AI 最近有多火”，那它可能不会让你觉得特别热闹。但如果你真正想把 Codex 纳入自己的开发流程，想少走一些试错弯路，想尽快从“知道它”走到“用好它”，那么这本书会很适合你。

希望你读完之后，得到的不是一种短暂的新鲜感，而是一套能长期复用的方法。
