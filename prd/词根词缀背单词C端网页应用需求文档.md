# 词根词缀背单词 C 端网页应用需求文档

## 1. 文档概述

### 1.1 产品名称

词根词缀背单词

### 1.2 产品定位

一款面向普通英语学习用户的 C 端网页应用，通过“词根 + 词缀 + 词族联想 + 例句记忆”的方式，帮助用户更高效地理解、记忆和复习英语单词。

### 1.3 产品目标

- 让用户不再依赖纯机械重复背单词
- 用词根词缀规律帮助用户建立词汇网络
- 提供适合碎片化使用的移动端体验
- 在网页端实现轻量、低门槛、可持续使用的背词产品

### 1.4 目标平台

- Web 优先
- 兼容桌面浏览器
- 重点兼容移动端 H5 页面体验

## 2. 背景与机会

### 2.1 当前问题

现有背单词产品大多有以下问题：

- 更强调刷词数量，较少强调构词规律理解
- 单词之间缺乏联系，用户容易遗忘
- 记忆方式偏机械，复习体验单调
- 许多产品功能过重，对轻量用户不友好
- 在移动端虽然可用，但并未针对碎片化学习做足够优化

### 2.2 产品机会

词根词缀天然适合做“知识关联型记忆”。如果把单词记忆从孤立卡片变成“构词关系网”，用户对词义猜测能力、记忆牢固度和复习效率都会明显提升。

## 3. 目标用户

### 3.1 核心用户

- 高中生和大学生
- 准备四六级、考研、雅思、托福等考试的用户
- 有英语学习需求的职场白领
- 想通过规律记忆提升词汇量的自学用户

### 3.2 用户特征

- 愿意使用手机进行碎片化学习
- 对传统单词卡片的遗忘率有明显感受
- 希望理解单词构成，而不是死记硬背
- 对界面和学习节奏敏感，不喜欢复杂难上手产品

## 4. 产品目标与成功指标

### 4.1 产品目标

- 提供基于词根词缀的结构化背词体验
- 帮助用户建立词汇之间的关联记忆
- 实现每日学习和复习闭环
- 提供适配移动端的高频使用体验

### 4.2 核心指标

- 新用户首日完成首轮学习率
- 次日留存率
- 7 日留存率
- 人均每日学习单词数
- 人均每日复习完成率
- 收藏词根词缀卡片的使用率
- 词义测试正确率

## 5. 核心使用场景

### 5.1 每日碎片化学习

用户在通勤、排队、休息等碎片时间打开网页，快速学习一组单词，并通过词根词缀理解记忆规律。

### 5.2 主题学习

用户主动选择某个词根或词缀，例如 `spect`、`tele`、`sub`，集中学习这一组词的含义和相关单词。

### 5.3 复习回顾

用户根据系统安排进行复习，通过拼写、释义、选择题或词义联想题巩固已学内容。

### 5.4 临考强化

用户按考试目标词库进行重点学习，快速掌握高频词根词缀和相关词汇。

## 6. 产品范围

### 6.1 本期范围

- 注册登录或游客模式
- 词根词缀卡片学习
- 单词详情页
- 每日学习计划
- 复习计划与简单遗忘曲线
- 收藏与错词本
- 搜索功能
- 移动端适配

### 6.2 暂不纳入本期

- 社区功能
- 多人打卡 PK
- 真人直播课程
- 复杂游戏化成长体系
- 原生 App

## 7. 核心功能需求

### 7.1 首页

首页需承载三个目标：

- 让新用户快速理解产品价值
- 让老用户快速进入今日学习
- 提供词根词缀探索入口

首页建议包含：

- 顶部品牌区与搜索框
- 今日学习进度卡片
- 今日待复习数量
- 推荐词根词缀专题
- 高频词根词缀入口
- 最近学习记录

### 7.2 词根词缀学习页

该页面是产品核心。

每张学习卡片应至少展示：

- 词根或词缀本体
- 基本含义
- 常见变体
- 相关联想图示或简短说明
- 代表单词列表
- 每个单词的简要释义

用户操作包括：

- 左右切换学习卡片
- 收藏词根词缀
- 标记“已理解”
- 查看相关词族

### 7.3 单词详情页

单词详情页需要承接“从词根到单词”的理解闭环。

每个单词应展示：

- 单词
- 音标
- 发音按钮
- 中文释义
- 英文释义
- 例句
- 构词拆解
- 对应词根和词缀解释
- 同词根相关词推荐

### 7.4 学习模式

系统至少提供两种学习模式：

- 按词根词缀学习
- 按考试词表学习

后续可支持：

- 按主题词汇学习
- 按难度分层学习

### 7.5 复习模式

复习页用于提升留存和学习效果。

复习题型建议包含：

- 看词选义
- 看义选词
- 判断词根含义
- 构词拆解判断

系统需要记录：

- 用户是否答对
- 错误次数
- 最近复习时间
- 下次推荐复习时间

### 7.6 搜索功能

用户可以搜索：

- 单词
- 词根
- 词缀

搜索结果应支持快速跳转到：

- 单词详情页
- 词根词缀卡片页
- 相关词族列表

### 7.7 收藏与错词本

用户可收藏：

- 词根词缀
- 单词

系统应自动生成错词本，用于复习优先级排序。

错词本页面应支持：

- 按最近出错排序
- 按错误次数排序
- 按词根分组查看

### 7.8 学习计划

用户首次进入产品时，系统可引导设置：

- 学习目标
- 每日学习词量
- 考试类型
- 每日提醒时间

系统应基于用户设置生成：

- 今日学习单
- 今日复习单
- 每周学习概览

## 8. 内容与数据需求

### 8.1 词根词缀内容结构

每个词根词缀条目建议包含：

- 编号
- 词根或词缀本体
- 基础含义
- 扩展含义
- 常见位置
- 记忆提示
- 相关单词列表

### 8.2 单词内容结构

每个单词条目建议包含：

- 单词
- 音标
- 发音地址
- 中文释义
- 英文释义
- 词性
- 例句
- 词根拆解
- 所属考试级别
- 难度等级
- 相关近义词或同词族词

## 9. 用户流程

### 9.1 新用户流程

1. 进入首页
2. 了解产品价值
3. 选择游客体验或登录
4. 设置学习目标
5. 开始第一组词根词缀学习
6. 完成首次复习体验

### 9.2 老用户流程

1. 打开首页
2. 查看今日待学习与待复习数量
3. 进入学习或复习
4. 查看结果反馈
5. 收藏重点内容或进入错词本

建议第一版包含以下页面：

- 首页
- 登录/注册页
- 学习页
- 单词详情页
- 复习页
- 搜索页
- 收藏页
- 错词本页
- 个人中心页

移动端底部导航建议包含：

- 首页
- 学习
- 复习
- 收藏
- 我的

## 11. 移动端适配要求

本产品需重点兼容移动端使用场景。

### 11.1 设计原则

- 单手可操作
- 信息层级清晰
- 点击区域足够大
- 适合碎片化阅读和快速切换
- 减少复杂悬浮交互

### 11.2 页面要求

- 首页首屏需在手机上快速展示“今日学习入口”
- 学习卡片支持上下滚动和左右切换的自然体验
- 重要按钮固定在拇指友好区域
- 字体大小和行距适合小屏阅读
- 音频播放、收藏、切换等高频操作需一键可达

### 11.3 兼容要求

- 支持主流移动浏览器
- 支持 iPhone 和主流 Android 屏幕尺寸
- 页面在 375px 到 430px 宽度区间保持良好布局
- 桌面端需做自适应扩展，而不是简单放大移动端界面

## 12. 视觉与交互要求

### 12.1 视觉方向

- 年轻化但不过度幼稚
- 有学习产品的可信感
- 突出“规律记忆”和“知识网络”感
- 保持干净、轻盈、可持续使用的视觉体验

### 12.2 交互要求

- 学习反馈明确
- 卡片切换流畅
- 复习结果即时反馈
- 收藏和错词操作低成本
- 加载和提交状态要有明显提示

## 13. 商业化与增长建议

### 13.1 增长方向

- 游客可体验部分内容
- 登录后保存学习记录
- 分享词根词缀卡片到社交平台
- 连续学习天数激励
- 通过考试专题页获取自然搜索流量

### 13.2 商业化方向

- 免费基础功能 + 高级会员
- 高级会员解锁完整词库、专项词表、深度复习报告
- 考试专题包付费
- 高质量音频或讲解包付费

## 14. 非功能需求

### 14.1 性能

- 首页首屏加载快
- 移动端低端机也可流畅切换学习卡片
- 搜索响应及时

### 14.2 可用性

- 新用户 3 分钟内可以开始第一次学习
- 主要路径不超过 3 次点击进入学习页

### 14.3 数据安全

- 用户学习记录可靠保存
- 登录与账户信息具备基础安全保护

## 15. MVP 建议

第一版 MVP 建议只做以下能力：

- 首页
- 按词根词缀学习
- 单词详情页
- 每日学习与复习
- 收藏与错词本
- 搜索
- 移动端适配

先不做：

- 社区互动
- 多人竞争
- 高复杂度成长体系
- 过多视觉特效

## 16. 里程碑建议

### Phase 1：内容验证

- 搭建基础词根词缀内容库
- 上线基础学习页和详情页
- 验证用户是否认可“规律记忆”方式

### Phase 2：学习闭环

- 增加复习机制
- 增加错词本和收藏
- 增加考试词表切换

### Phase 3：增长与商业化

- 增加会员能力
- 增加专题词库
- 增加分享和增长组件

## 17. 风险与难点

- 词根词缀内容质量决定产品上限
- 如果学习体验太像传统单词卡片，会丧失差异化
- 移动端如果信息过多，用户会快速疲劳
- 复习机制如果设计不合理，会影响留存
