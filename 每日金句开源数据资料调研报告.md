# 每日金句开源数据资料调研报告

更新时间：2026-03-21

## 1. 背景

当前需求是围绕以下输入信息，生成“每日金句”能力：

- 用户基础信息
- 今日运势
- 近七天对话核心记忆
- 当日用户安排事项

目标不是单纯随机返回一句名言，而是形成一个可用于产品落地的“个性化每日金句推荐”能力。为此，需要先确定适合接入的开源 GitHub 数据资料或 API，作为金句语料、运势补充数据以及中文表达增强的数据底座。

## 2. 结论先行

推荐采用“三层数据源”方案：

1. 主金句数据源首选 `Quotable`
2. 运势补充数据源可选 `aztro`
3. 中文气质增强数据源建议补充 `Chinese-Poetry-Dataset`

如果只选一个最实用、最容易落地的开源 GitHub 数据资料，优先级最高的是：

- `Quotable`

原因如下：

- 开源成熟度较高，社区接受度较好
- 提供现成 API 与独立数据仓库两种接入方式
- 支持按作者、标签、长度等维度筛选
- 适合做“每日金句”的召回层和基础语料层
- 可以先直接接 API，后续再切换为本地落库自管

## 3. 候选数据资料分析

## 3.1 `Quotable`

GitHub：

- https://github.com/lukePeavey/quotable
- https://github.com/quotable-io/data

定位：

- 开源名言 API 与对应数据仓库

适合原因：

- 提供随机名言、按条件过滤、按作者查询等基础能力
- 适合做“每日一句”“场景化推荐”“按用户标签推荐”
- 数据结构相对规整，便于本地清洗和二次加工
- 数据仓库中有 `data` 与 `translations` 目录，适合后续多语言或本地化扩展

工程价值：

- 可以快速做 MVP
- 可以作为推荐系统中的候选池
- 可以与用户画像、情绪标签、日程压力等级、对话记忆标签联动

适用场景：

- 每日一句
- 励志类推荐
- 工作日程前置激励
- 结合“今日运势”输出更有仪式感的文案

风险与边界：

- 语料整体偏英文互联网常见名言风格
- 若直接机翻，中文质感可能一般
- 不适合作为唯一中文内容源

结论：

- 作为主数据源，推荐度最高

## 3.2 `aztro`

GitHub：

- https://github.com/sameerkumar18/aztro

定位：

- 开源星座运势 API

适合原因：

- 可以提供 `today`、`tomorrow`、`yesterday` 维度的日运势
- 包含每日描述、幸运数字、幸运时间、情绪、颜色、匹配星座等字段
- 适合给“每日金句”增加一点运势上下文

工程价值：

- 适合做轻量级“今日运势 + 金句”产品
- 可以在没有自建命理引擎时快速补足运势字段

风险与边界：

- 本质上是西方星座运势接口，不等于完整命理系统
- 若产品未来要做更严肃的中式命理或混合命理，需替换或补强
- 只能作为运势辅助层，不能替代金句主语料

结论：

- 适合作为辅助数据源，不适合作为核心内容源

## 3.3 `Chinese-Poetry-Dataset`

GitHub：

- https://github.com/hlthu/Chinese-Poetry-Dataset

定位：

- 中文古典诗词数据集

适合原因：

- 能显著提升中文表达气质
- 适合在“每日金句”之外扩展出“今日一句”“诗意一句”“东方表达版每日句子”
- 对中文用户更自然，不会有强烈翻译腔

工程价值：

- 可和现代名言库混用，做不同风格路由
- 适合节气、黄历、早安晚安、旅行、独处、治愈等场景

风险与边界：

- 不是现代名人名言库
- 更适合做诗意表达，而不是标准的“作者-名言”型卡片
- 需要额外做筛选，避免语义过古或场景不贴合

结论：

- 很适合作为中文风格增强层

## 4. 推荐方案

## 4.1 最小可行方案

仅接入：

- `Quotable`

用途：

- 先跑通“每日金句”核心功能
- 每天根据用户状态从候选池中选一句

适合阶段：

- 原型验证
- 小规模上线
- 先做推荐逻辑，再补内容深度

## 4.2 实用上线方案

组合接入：

- `Quotable` + `aztro`

用途：

- `Quotable` 提供金句主语料
- `aztro` 提供今日运势上下文

适合阶段：

- 希望产品有“每日陪伴感”
- 希望输出不是纯随机，而是与当天状态有弱相关

## 4.3 中文体验优化方案

组合接入：

- `Quotable` + `aztro` + `Chinese-Poetry-Dataset`

用途：

- `Quotable` 负责现代名言主语料
- `aztro` 负责今日运势标签
- `Chinese-Poetry-Dataset` 负责中文审美增强

适合阶段：

- 面向中文用户的正式产品
- 对文案气质和表达质量有更高要求

## 5. 面向产品落地的使用建议

建议把“每日金句”能力拆成三层：

### 5.1 数据层

- 金句语料库
- 运势数据
- 中文诗句或中文表达增强语料

### 5.2 召回与筛选层

根据以下信息做候选召回：

- 用户基础信息
- 今日运势标签
- 近七天对话记忆关键词
- 当日安排事项标签

示例标签包括：

- 工作压力高
- 适合鼓励
- 适合稳定情绪
- 适合执行力提醒
- 适合治愈陪伴
- 适合旅行或出行语境

### 5.3 生成层

最终输出不一定直接照搬原句，可以采用以下方式：

- 直接返回原始金句
- 返回原句 + 一句系统解释
- 返回原句 + 结合运势与日程的个性化导语

示例输出结构：

1. 今日金句
2. 作者
3. 推荐理由
4. 与今日安排的关联说明

## 6. 最终建议

综合开源成熟度、工程可接入性和产品实用性，推荐顺序如下：

1. 第一优先级：`Quotable`
2. 第二优先级：`aztro`
3. 第三优先级：`Chinese-Poetry-Dataset`

最终判断：

- 如果现在要快速落地一个“每日金句”功能，先接 `Quotable`
- 如果要把“今日运势”纳入推荐逻辑，补接 `aztro`
- 如果要提升中文表达质感，再补 `Chinese-Poetry-Dataset`

因此，本次调研的推荐结论是：

- 主数据资料选用 `Quotable`
- 运势辅助资料选用 `aztro`
- 中文风格增强资料选用 `Chinese-Poetry-Dataset`
